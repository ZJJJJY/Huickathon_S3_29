# 三分钟热度 · 最小设计 (design_minimal.md)

> 这是相对 `design.md` 的精简版，锁定经过对齐后的最小决策。
> 报告的具体结构留空，待后续确定后再补。

---

## 1. 项目一句话

用户在被某个爱好击中时，通过简短问卷 + 从精选清单里挑一个爱好，30 秒内得到一份结构化「入坑决策报告」。

---

## 2. 已对齐的最小决策

### 2.1 爱好分类（6 类）

| 分类 | 示例 |
|---|---|
| 技能成长型 | 吉他、第二外语 |
| 体能体验型 | 滑雪、飞盘 |
| 创作手工型 | 陶艺、木工 |
| 收藏鉴赏型 | 手办、球鞋 |
| 自然户外型 | 野钓、露营 |
| 社交竞技型 | 羽毛球、剧本杀 |

每个分类下准备 **4–5 个候选爱好**，由人手工填写清单（不做自由输入）。同时保留「🎲 随便给我来一个」兜底。

### 2.2 内容生成方式

**完全离线预跑。** 每个爱好的结构化报告通过 agent prompt + tool use 提前产出，固化为 JSON 文件。运行时后端**不调 LLM**，只读静态文件。

带来的简化：
- 不需要 SSE 流式
- 不需要 Serper / Tavily 主备
- 不需要 JSON 修复 / 降级链路
- 不需要 Tool Use 编排在线运行

### 2.3 用户流程

```
扫码落地页 → 问卷（题目可改）→ 分类+爱好选择页 → 报告页
```

问卷依旧前置，但具体题目和字段可后续调整。报告结构留空。

### 2.4 UI 调性

- 参考方向：**B 站 / 抖音风格**，深色底 + 霓虹强调色
- 选爱好页**不做滑动**，采用「分类分组 + 卡片入场动画 + 选中态霓虹边框」
- 报告页用前端渲染节奏（逐卡片淡入 / 打字机 / 骨架屏）模拟"AI 正在工作"的感觉，但实际是静态数据

### 2.5 端形态

仅 **Mobile-First Web**，不考虑桌面端。

## 4. 显式不做的事

- 用户登录 / 跨设备同步
- 数据库
- 直接爬抖音 / 小红书
- 图片内容理解
- PC 端适配
- 在线 Agent 调用、SSE 流、搜索 API
- 报告底部对话式追问
- 地图 API
